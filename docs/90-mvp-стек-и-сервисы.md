# MVP: стек и сервисы (заменяемый слой)

Этот документ описывает **текущие** решения по инструментам для MVP.
Если стек будет меняться — концептуальный слой в `01–06` должен остаться валиден.

## Backend
- **Django**: базовый веб-фреймворк.
- **Django REST Framework (DRF)**: API для читалки и менеджерки.

## База данных
- **PostgreSQL**: основное хранилище моделей контента, задач, статусов.

## Очередь и фоновые задачи
- **Celery**: воркеры, ретраи, цепочки задач (пайплайн).
- **Redis**: брокер/бэкенд для Celery на MVP (возможна замена на RabbitMQ позже).

## Медиа-хранилище
- **S3-совместимое хранилище**:
  - MVP/dev: **SeaweedFS (S3 gateway)**
  - prod: AWS S3 / Cloudflare R2 / любой S3 compatible

## Реалтайм/обновления статусов
- MVP: **polling** из менеджерского UI (периодический запрос статусов).
- Позже: **WebSocket** (например, через Django Channels) для live-обновлений.

## Frontend
- Читалка: **Vue** (или Nuxt).
- Менеджерка на MVP: **Django Admin + кастомные admin views** (страница Review + Preview).
- Позже: вынос менеджерки в отдельный **Vue SPA** поверх тех же API.

## Интеграция с AI сервисами
Абстракция “провайдер”:
- вход: scene card + стиль + параметры формата,
- выход: asset (image/video) + метаданные,
чтобы можно было сменить поставщика без перелопачивания модели данных.
