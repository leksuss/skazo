# MVP: схема экранов и API (заменяемый слой)

Цель MVP — закрыть полный цикл “txt → сцены → иллюстрации → ревью → публикация → чтение”.

## Экраны менеджера (минимум)

### 1) Import (создание черновика)
- поля: название, язык, возраст/категория, теги
- загрузка: `.txt`
- стиль: выбрать пресет или загрузить референс
- кнопка: “Запустить обработку”

### 2) Review (сцены + иллюстрации)
- список сцен:
  - текст сцены / блоки
  - текущая иллюстрация (или несколько вариантов)
  - статусы генерации
- действия:
  - “перегенерировать сцену”
  - “изменить описание сцены (scene card)”
  - “сдвинуть якорь” (если якорь привязан к блоку)
  - “принять / отклонить”

### 3) Preview (как книга)
- переключатели:
  - phone / tablet
  - portrait / landscape
  - размер шрифта
- перелистывание (рендер с клиентской пагинацией)

### 4) Publish
- кнопка “Опубликовать версию”

## API (в общих чертах)

### Story
- создать/обновить черновик
- получить текущую версию и её статус

### Version / Structure
- получить сцены/блоки/якоря
- обновить: якорь/scene card/текст блока (с созданием новой версии или с черновой правкой — по выбранной политике)

### Tasks
- запустить пайплайн обработки
- запустить перегенерацию по сцене/якорю
- получить статусы задач (для polling)

### Reader
- получить опубликованную версию (структура + ассеты)

Примечание: конкретные URL/методы и форматы — реализационная деталь и появятся при проектировании API.
