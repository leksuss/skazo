# Документация: Skazo (рассказы и сказки)

Эта папка специально разделена на два слоя:

- **Концепции** — стабильные идеи и принципы, которые сохраняются даже если менять стек/инструменты.
- **MVP / Технологии** — конкретные решения “как сделать сейчас”; их можно переработать, не трогая концептуальный слой.

## Как читать

### Управление проектом (для разработки с помощью LLM)
- `ROADMAP.md` — итерации/эпики/задачи, зависимости и критерии готовности (DoD).
- `DECISIONS.md` — журнал ключевых решений (ADR), чтобы не “дрейфовать” со временем.
  - Дефолтная команда для управления разработкой короткими сообщениями описана в `ROADMAP.md` (раздел “Дефолтная команда”).

### Концепции (стабильный слой)
- `01-концепции-и-границы.md` — что именно строим, ключевые принципы, не-цели.
- `02-модель-контента.md` — каноническая модель текста, сцены, якоря, версии.
- `03-пайплайн-подготовки-контента.md` — этапы: импорт → нормализация → структурирование → ассеты → ревью → публикация.
- `04-читалка-и-пагинация.md` — “книга”, разворот/страница, почему пагинация на клиенте.
- `05-ревью-и-модерация.md` — как менеджер принимает/перегенерирует, что автоматизировать.
- `07-style-presets.md` — как хранить стили (референсы + style bible), версионирование и влияние на генерацию.
- `06-глоссарий.md` — термины.

### MVP / Технологии (заменяемый слой)
- `90-mvp-стек-и-сервисы.md` — Django/DRF, Postgres, Celery+Redis, S3/MinIO и т.п.
- `91-mvp-схема-экранов-и-эндпойнтов.md` — минимальный набор экранов и API для старта.
- `92-mvp-очереди-и-состояния.md` — типы задач, статусы, идемпотентность, ретраи.
- `93-mvp-практичная-архитектура.md` — “приземление” на сервисы и их ответственность: как это будет жить в dev/prod.
